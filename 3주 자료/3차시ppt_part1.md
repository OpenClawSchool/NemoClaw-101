# 3차시. 실전 워크플로우 설계 — 슬라이드

## 📊 소스 분석 요약
- 핵심 주제: 실행형 AI 워크플로우 설계, 역할별 에이전트 시나리오, 자동화 기준
- 핵심 메시지: 에이전트의 가치는 기능이 아니라 업무 흐름 안에 올바르게 배치하는 설계에서 나온다
- 대상 청중: NemoClaw 교육 수강생 (개인 생산성, 창업가, 전문가, 개발자)
- 포맷: Presenter Slides
- 총 슬라이드 수: 22장 → 2개 파일로 분할
- 예상 발표 시간: 35분 (Part 1: 22분 / Part 2: 13분)

## 📂 파일 구성
| 파일 | 슬라이드 | 내용 |
|------|----------|------|
| **Part 1 (이 파일)** | 1~14장 | 설계 원칙·프레임, 개인 비서·창업가 비서 시나리오 |
| Part 2 | 15~22장 | 전문가·개발자 시나리오, 자동화 기준, 실습, 핵심 정리 |

## 📝 슬라이드 스크립트

---
### 슬라이드 1: 타이틀

**슬라이드 내용**
- 3차시. 실전 워크플로우 설계
- 개인 비서 · 창업가 비서 · 전문가 에이전트
- 기술 이해에서 업무 설계로

**시각적 제안**
- 레이아웃: 중앙 정렬, 대형 타이틀
- 배경: 소프트 블루-화이트 그라디언트
- 아이콘: 입력→판단→실행→승인→기록 5단계 노드

---
### 슬라이드 2: 전체 목차

**슬라이드 내용**
- Part 1 — 설계 원칙과 시나리오 1·2
  - 워크플로우 정의 / 5원칙 / 프레임 / 개인 비서 / 창업가 비서
- Part 2 — 시나리오 3·4와 실습
  - 전문가 에이전트 / 개발자 보조 / 자동화 기준 / 실습

**시각적 제안**
- 레이아웃: 2열, Part 1(블루) / Part 2(오렌지) 색상 구분
- 현재 위치: Part 1 강조 표시

---
### 슬라이드 3: 왜 이번 차시가 핵심인가

**슬라이드 내용**
- 구조를 이해해도 실무 적용은 별개
- 현장 질문:
  - 무엇을 에이전트에 맡길 수 있는가
  - 어디까지 자동화할 수 있는가
  - 어디서 사람이 검토해야 하는가
- 핵심: 기능이 아닌 배치가 가치를 만든다

**시각적 제안**
- 레이아웃: 좌측 질문 3개, 우측 핵심 메시지 카드
- 강조: "배치"를 오렌지 태그로 표시

---
### 슬라이드 4: 학습 목표

**슬라이드 내용**
- 업무를 작업이 아닌 흐름으로 본다
- 4가지 역할 시나리오를 설계한다
- 입력-판단-실행-승인-기록 단계를 구분한다
- 자동화 영역과 사람 검토 영역을 나눈다

**시각적 제안**
- 레이아웃: 2×2 목표 카드 그리드
- 하단 배지: "오늘 끝나면 첫 에이전트 정의 가능"

---
### 슬라이드 5: 워크플로우 설계란 무엇인가

**슬라이드 내용**
- "메일 정리"는 기능이 아니라 흐름
- 실제 흐름: 읽기 → 분류 → 추출 → 초안 → 검토 → 발송 → 기록
- 정의: 업무를 5단계 흐름으로 재구성하는 작업

**시각적 제안**
- 레이아웃: 수평 7단계 화살표 프로세스
- 강조: 검토·승인 단계에 사람 아이콘

---
### 슬라이드 6: 좋은 워크플로우 5원칙

**슬라이드 내용**
- 1) 작고 명확한 문제부터 시작
- 2) 읽기 중심 자동화부터 적용
- 3) 승인 포인트를 반드시 설계
- 4) 기록이 남는 구조 유지
- 5) 사람 역할은 제거가 아닌 전환

**시각적 제안**
- 레이아웃: 세로 스택 체크리스트, 번호 배지
- 색상: 각 항목 강조 배지 미묘하게 구분

---
### 슬라이드 7: 워크플로우 설계 프레임 5단계

**슬라이드 내용**
| 단계 | 내용 | 예시 |
|---|---|---|
| 입력 | 무엇을 받는가 | 메일, 일정, 문서 |
| 판단 | 어떤 기준으로 분류 | 중요도, 긴급성 |
| 실행 | 무엇을 수행하는가 | 요약, 초안 작성 |
| 승인 | 사람이 어디서 개입 | 발송 전, 공유 전 |
| 기록 | 결과를 어디에 남김 | 노트, 로그, CRM |

**시각적 제안**
- 레이아웃: 오각형 사이클 다이어그램
- 전환: "이 프레임을 시나리오에 바로 적용합니다"

---
### 슬라이드 8: 시나리오 1 — 개인 생산성 비서 개요

**슬라이드 내용**
- 왜 유용한가: 판단보다 정리 비용을 줄여준다
- 대표 업무:
  - 오늘 일정 요약
  - 중요 메일 선별
  - 할 일 재정리
  - 문서 검색

**시각적 제안**
- 레이아웃: 좌측 유용성 메시지, 우측 업무 목록 카드
- 강조: "앱을 덜 열고, 판단에 집중"

---
### 슬라이드 9: 아침 브리핑 에이전트

**슬라이드 내용**
- 입력: 캘린더, 읽지 않은 메일, 미완료 작업
- 판단: 일정 정렬, 긴급 메일 식별, 오늘 우선순위
- 실행: 브리핑 생성, 충돌 표시, 메일 요약
- 승인: 읽기 중심 → 승인 없이 제공 가능
- 기록: 브리핑 파일 저장, 일일 노트 기록

**시각적 제안**
- 레이아웃: 5단계 수직 플로우 카드
- 강조: 승인 단계에 "읽기 전용" 배지

---
### 슬라이드 10: 답장 보조 에이전트

**슬라이드 내용**
- 입력: 새 메일, 응답 스타일, 기존 템플릿
- 판단: 답장 필요 여부 / 즉답 가능 여부
- 실행: 3줄 요약 + 응답 초안 + 보류/즉답/중요 분류
- 승인: 실제 발송은 사람 승인 후

**시각적 제안**
- 레이아웃: 좌측 메일 카드, 우측 에이전트 결과 카드
- 강조: 발송 단계에 ⚠️ 아이콘

---
### 슬라이드 11: 시나리오 1 — 자동화 vs 사람 검토

**슬라이드 내용**
| 자동화 가능 | 사람 검토 필요 |
|---|---|
| 일정 요약 | 실제 회신 발송 |
| 메일 요약 | 중요한 약속 확정 |
| 할 일 분류 | 외부 공유 |
| 문서 검색 | 일정 변경 확정 |

**시각적 제안**
- 레이아웃: 좌우 대비 표 (초록 vs 오렌지)
- 하단 원칙: "읽기는 넓게, 쓰기는 좁게"

---
### 슬라이드 12: 시나리오 2 — 창업가 운영 비서 개요

**슬라이드 내용**
- 왜 유용한가: 생산보다 운영 정리에 시간을 낭비
- 대표 업무:
  - 고객 문의 분류
  - 미팅 준비 자료 정리
  - 제안서 초안 구성
  - 후속 연락 체크리스트

**시각적 제안**
- 레이아웃: 상단 유용성, 하단 업무 4개 카드
- 강조: "운영 업무를 채팅으로 압축"

---
### 슬라이드 13: 고객 문의 정리 에이전트

**슬라이드 내용**
- 입력: 이메일/채팅 문의, FAQ, 응답 템플릿
- 판단: 유형 분류, 긴급성, 영업/지원/제휴 구분
- 실행: 요약 + 태깅 + 응답 초안 + 처리 순서
- 승인: 고객 발송은 반드시 승인 후
- 기록: 문의 로그, CRM 후보, 후속 연락 목록

**시각적 제안**
- 레이아웃: 파이프라인 보드 (유입→분류→초안→승인)
- 색상: 단계별 진행 상태 표시

---
### 슬라이드 14: Part 1 요약 + Part 2 예고

**슬라이드 내용**
- 설계는 기능이 아닌 흐름에서 시작한다
- 5단계 프레임이 모든 시나리오의 공통 구조다
- 개인/창업가 비서는 읽기+초안 자동화에 효과적
- 다음: 전문가·개발자 시나리오 + 자동화 기준 + 실습

**시각적 제안**
- 레이아웃: 좌측 3줄 요약, 우측 Part 2 미리보기 카드
- 전환: "이제 역할별 고도화와 운영 기준으로 넘어갑니다"
