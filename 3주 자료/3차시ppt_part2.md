### 슬라이드 17: 상담 전 브리핑 에이전트

**슬라이드 내용**

**시각적 제안**
- 레이아웃: 5단계 수직 플로우 카드
- 강조: "전문성을 저장이 아닌 흐름으로"

### 슬라이드 18: 시나리오 3 — 자동화 vs 사람 검토

**슬라이드 내용**
| 자동화 가능 | 사람 검토 필요 |
|---|---|
| 자료 검색 | 맞춤 조언 최종 확정 |
| 상담 준비 브리핑 | 계약/법률/재무 답변 |
| FAQ 정리 | 외부 발송 |
| 콘텐츠 재정리 | 고객별 특수 상황 판단 |

**시각적 제안**
- 레이아웃: 좌우 대비 표 (초록 vs 오렌지)
- 하단 원칙: "전문성은 자동화, 책임은 사람이 통제"

### 슬라이드 19: 시나리오 4 — 개발자/운영 보조 에이전트

**슬라이드 내용**
- 왜 유용한가: 분산된 정보를 채팅으로 모아 우선순위 표시
- 이슈 브리핑: 긴급도 분류 → 담당자 후보 → 중복 묶기
- 장애 대응: 상황 요약 → 의심 원인 → 점검 체크리스트
- 실제 배포·설정 변경은 승인 후 실행

**시각적 제안**
- 레이아웃: 2개 플로우 카드 (이슈 브리핑 / 장애 대응)
- 강조: 배포·변경 단계에 ⚠️ 승인 아이콘

### 슬라이드 20: 자동화 적합 vs 사람 검토 기준

**슬라이드 내용**
| 자동화 적합 | 사람 검토 필요 |
|---|---|
| 반복적이다 | 외부 발송 포함 |
| 입력 형식이 일정하다 | 실제 변경 발생 |
| 읽기 중심이다 | 금전/계약/법적 책임 |
| 최종 검토가 쉽다 | 되돌리기 비용 크다 |

**시각적 제안**
- 레이아웃: 좌우 매트릭스, 초록 vs 오렌지
- 강조: "무인화가 아니라 검토 가능성이 목표"

### 슬라이드 22: Key Takeaways + 다음 차시

**슬라이드 내용**
- 1) 에이전트 가치는 워크플로우 설계에서 나온다
- 2) 시작은 작고 반복적인 읽기 업무
- 3) 5단계 프레임은 모든 역할에 적용 가능
- 4) 자동화는 검토 가능하게 설계해야 지속 가능
- 5) 사람 역할은 제거가 아닌 전환
- 다음 차시: 안전한 자동화 체계와 운영 정책

**시각적 제안**
- 레이아웃: 좌측 5줄 요약 (번호+아이콘), 우측 4차시 예고 카드
- 마무리: "설계에서 운영으로, 이제 안전성을 더합니다"
