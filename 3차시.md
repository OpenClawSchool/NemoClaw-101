# 3차시. 실전 워크플로우 설계

## 학습 목표
- NemoClaw 위에서 어떤 업무가 잘 맞는지 구체적으로 설계할 수 있다.
- “채팅 중심 처리”의 장점과 한계를 이해한다.
- 개인용에서 팀용/전문가용으로 확장하는 방법을 그릴 수 있다.

## 핵심 메시지
- NemoClaw는 기능 과시용이 아니라, 반복적이고 디지털 흔적이 많고, 승인 흐름을 넣기 쉬운 업무에 잘 맞습니다.
- 특히 OpenClaw의 장점인 채팅 인터페이스 + 실행 + 메모리 + 자동화를 유지하면서, 위험한 실행은 더 통제된 방식으로 다루게 해 줍니다.

## 60분 진행안

### 0~10분: 지난 시간 리뷰
- “구조 이해”에서 “업무 설계”로 전환

### 10~25분: 개인 생산성 비서 시나리오
- 아침 일정 브리핑, 메일 triage, 회신 초안, 해야 할 일 추출, 저녁 회고 리포트

### 25~35분: 창업가/1인기업 운영 비서 시나리오
- 고객 문의 분류, 미팅 전 자료 정리, 문서 초안 작성, CRM 업데이트 후보 생성, 공지/후속 연락 초안

### 35~45분: 전문가 에이전트 시나리오
- Obsidian/문서/노트 기반 응답, 고객별 맞춤 응답 초안, 강의 전 브리핑, 지식 자산 Skills화

### 45~55분: 어떤 업무가 자동화에 적합한가
- **적합**: 분류, 요약, 초안, 리마인드, 조회, 정리
- **승인 필요**: 발송, 삭제, 외부 공개, 결제, 권한 변경
- **부적합**: 법적 확정, 고위험 의료/재무 의사결정, 무제한 자율 실행

### 55~60분: 본인용 워크플로우 1개 선택

## 실습
아래 중 하나를 골라 워크플로우 설계:
1. **개인 비서형**: 일정 요약, 메일 분류, 회신 초안, 승인 요청
2. **창업 운영형**: 문의 모니터링, 우선순위 분류, 답변 초안, CRM 기록
3. **전문가 에이전트형**: 자료 검색, 노트 묶음, 상담 전 브리핑, 후속 메일 초안

## 산출물
- 내 업무용 대표 워크플로우 1개
- 자동/승인/금지 포인트가 표시된 흐름도

## 과제
현재 하고 있는 반복 업무 1개를 아래 구조로 정리:
1. 입력
2. 처리
3. 출력
4. 승인 단계
5. 실패 시 대응
