### 슬라이드 6: 샌드박스 — 있을 때 vs 없을 때

**슬라이드 내용**
| | 없이 실행 | 안에서 실행 |
|---|---|---|
| 파일 | 전체 접근 | /sandbox만 |
| 네트워크 | 어디든 접속 | 허용 주소만 |
| API 키 | 직접 사용 | 호스트에 보관 |
| 사고 범위 | 시스템 전체 | 샌드박스 내 |

**시각적 제안**
- 레이아웃: 좌우 비교, 좌측(빨강 ⚠️) vs 우측(초록 ✅)
- 하단 한 줄: "통제를 시작할 수 있게 만드는 최소 단위"

### 슬라이드 7: 프롬프트 vs 정책

**슬라이드 내용**
- 프롬프트 = "이 방에 들어가지 마"라고 **말하는** 것
- 정책 = 애초에 **문을 잠그는** 것
- 프롬프트로는 네트워크 차단, 실행 권한 제한 불가

**시각적 제안**
- 레이아웃: 좌우 대비 (말풍선 vs 자물쇠 아이콘)
- 강조: 정책 쪽에 체크마크, 프롬프트 쪽에 ✗ 표시

**슬라이드 내용**
