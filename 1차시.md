# 1차시. NemoClaw가 무엇이고 왜 필요한가

## 학습 목표
- OpenClaw와 NemoClaw의 관계를 설명할 수 있다.
- 일반 AI 챗봇과 실행형 에이전트의 차이를 이해한다.
- “왜 보안이 기능보다 먼저인가”를 이해한다.

## 핵심 메시지
- 일반 AI 챗봇은 질문에 답한다. OpenClaw는 질문에 답할 뿐 아니라 실행한다. NemoClaw는 그 실행을 더 통제된 경계 안에 넣는 도구다.
- 공식 문서 기준 NemoClaw는 OpenClaw 자체를 대체하는 제품이 아니라, NVIDIA OpenShell 기반 샌드박스에서 OpenClaw를 더 안전하게 돌리기 위한 플러그인/스택입니다.

## 60분 진행안

### 0~10분: 과정 소개
- OpenClaw와 NemoClaw의 한 줄 정의
- 왜 “기능 소개”보다 “운영 구조 이해”가 먼저인지 설명

### 10~25분: 일반 LLM 챗봇 vs OpenClaw vs NemoClaw 비교
- 대화형 인터페이스 + 도구 실행 + 메모리 + 자동화 + 멀티채널 구조 설명
- “답변형 AI”와 “실행형 AI”의 책임 차이 설명

### 25~40분: NemoClaw의 공식 포지션 설명
- OpenClaw plugin for NVIDIA OpenShell
- open source models 활용 가능
- secure environment / sandbox / policy 기반 실행 강조

### 40~50분: 실제 활용 예시 3개
1. 이메일 초안 작성은 자동, 발송은 승인
2. 외부 URL 요약은 허용, 파일 삭제는 차단
3. 매일 아침 일정 브리핑은 자동, 외부 웹훅 호출은 제한

### 50~60분: 토론: “내 업무 중 무엇이 자동화에 적합한가?”
- 과제 안내

## 실습
본인 업무를 기준으로 아래 3가지 분류표 작성:
1. 자동 실행 가능
2. 승인 후 실행
3. 자동화 금지

## 산출물

- OpenClaw vs NemoClaw 이해 메모 1장
