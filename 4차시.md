# 4차시. 안전한 자동화와 운영 체크리스트

## 학습 목표
- 프롬프트 인젝션, 과도한 권한, 잘못된 자동 실행 위험을 이해한다.
- 승인형 자동화 원칙을 세울 수 있다.
- NemoClaw 도입 체크리스트를 만들 수 있다.

## 핵심 메시지
- NemoClaw를 잘 쓴다는 것은 모든 걸 자동화하는 것이 아니라, 위험한 실행을 검토 가능하게 만드는 것입니다.
- 공식 문서가 alpha 상태와 샌드박스/정책 기반 실행을 강조하는 이유도, 실행형 에이전트의 위험을 기능보다 먼저 다뤄야 하기 때문입니다.

## 60분 진행안

### 0~15분: 실행형 에이전트의 대표 위험
- 외부 URL 기반 프롬프트 인젝션, 이메일 본문/첨부파일 유도, 과도한 파일 접근, 잘못된 자동 발송, 장기 세션의 맥락 오염

### 15~30분: 승인형 자동화 원칙
- 읽기와 쓰기 분리, 초안과 발송 분리, 검색과 삭제 분리, 요약과 외부 게시 분리, 고위험 작업 이중 확인

### 30~40분: 운영 정책 설계
- 연결 채널, 허용 스킬, 외부 접근 허용 도메인, 로그 기록 정책

### 40~50분: NemoClaw PoC 체크리스트 작성
- 테스트 계정, 샌드박스 환경, 최소 권한, 승인 단계, 실패 대응, 사람 검토 포인트

### 50~60분: 최종 발표
- 내 운영 원칙 선언문 작성

## 실습
“내 NemoClaw 운영 규칙 10개” 작성 예:
- 메일 발송은 자동 금지
- 파일 삭제 자동 금지
- 외부 링크 기반 실행은 승인 필요
- 웹훅 호출은 화이트리스트만 허용
- 고객 데이터 포함 문서는 외부 모델로 바로 전송 금지
