# 4차시. 비용 최적화(토큰 절감) 및 사내 데이터 연동(로컬 RAG)

## 주제: 로컬 데이터 기반의 RAG 파이프라인 구축 및 실무 운영 최적화

## 학습 목표

- 사내 기밀 데이터 유출 방지를 위해 로컬 환경에서 문서 청킹 및 벡터화를 통한 RAG 파이프라인을 구축한다.
- 티어(Tier) 기반 모델 라우팅을 적용하여 업무 복잡도에 따라 로컬/클라우드 모델을 할당하고 비용을 절감한다.
- 'Thinking' 모드 비활성화 등 토큰 최적화(Hardening) 기술을 적용해 응답 속도를 개선하고 비용을 아낀다.
- 시스템 환경 변경 시 `nemoclaw onboard`를 활용한 안전한 재구축 운영 지침을 숙지한다.

---

## 1. 로컬 RAG(Retrieval-Augmented Generation) 파이프라인

에이전트가 사내 규정이나 매뉴얼을 '공부'하게 하여 똑똑하게 만드는 기술입니다. 보안을 위해 모든 과정은 외부 유출 없이 로컬에서 진행됩니다.

### 1.1 RAG의 3단계 원리 (도서관 비유)
1. **문서 청킹(Chunking):** 두꺼운 책을 에이전트가 한입에 먹기 좋게 '포스트잇' 단위로 쪼개는 작업입니다.
2. **벡터화(Vectorization):** 쪼개진 포스트잇을 의미에 따라 '지도'상의 좌표로 변환합니다. (예: 보안 관련 내용은 1번 구역, 네트워크는 2번 구역)
3. **지식 증강:** 사용자가 질문하면 에이전트가 관련 포스트잇을 먼저 찾아서 읽고, 그 내용을 바탕으로 대답합니다.

> **보안 이점:** 사내 기밀 문서를 외부 AI 서버로 보내지 않고 우리 서버 안에서만 처리하므로 데이터 유출 위험이 없습니다.

---

## 2. 티어(Tier) 기반 모델 라우팅

모든 업무에 비싼 최고급 모델(GPT-4 등)을 쓸 필요는 없습니다. 업무의 난이도에 따라 모델을 적절히 배분하여 비용을 80% 이상 아낄 수 있습니다.

| 티어 | 구분 | 적합한 업무 | 사용 모델 예시 |
| :--- | :--- | :--- | :--- |
| **L1 티어** | 가볍고 빠른 모델 | 간단한 일상 대화, 텍스트 요약, 데이터 분류 | Qwen 2.5 32B, Llama 3.1 8B |
| **L2 티어** | 무겁고 정교한 모델 | 복잡한 인프라 장애 추론, 전문 코드 분석, 논리 설계 | GPT-4o, Claude 3.5 Sonnet |

- **라우팅 로직:** 에이전트가 요청을 받으면 먼저 난이도를 판단합니다. 쉬운 일은 로컬의 L1 모델이 처리하고, 어려운 일만 클라우드의 L2 모델로 토스합니다.

---

## 3. 토큰 최적화 (Hardening)

에이전트가 말을 길게 한다고 해서 항상 좋은 것은 아닙니다. 꼭 필요한 말만 하도록 다듬어 비용과 속도를 모두 잡습니다.

- **Thinking 모드 최적화:** 에이전트가 내부적으로 고민하는 과정('생각')을 출력하지 않도록 설정하여 불필요한 토큰 낭비를 막습니다.
- **프롬프트 경량화:** "친절하게 대답해 줘" 같은 불필요한 수식어를 제거하고, 지시 사항을 명확한 불릿 포인트로 전달하여 컨텍스트 효율을 높입니다.
- **응답 길이 제한:** 필요한 정보만 간결하게 출력하도록 페르소나를 조정합니다.

## 4. 시스템 운영 및 유지보수 지침

에이전트 시스템은 여러 기술이 얽혀 있어 개별적으로 수정하면 꼬이기 쉽습니다. '통합 관리'가 핵심입니다.

- **안전한 재구축:** 샌드박스 설정을 바꾸거나 모델을 교체했다면, 고민하지 말고 `nemoclaw onboard` 명령어를 다시 실행하세요. 전체 스택의 정합성을 한 번에 맞춰줍니다.
- **개별 업데이트 지양:** `openshell self-update` 처럼 검증되지 않은 개별 업데이트는 시스템 충돌을 야기할 수 있습니다. 항상 NemoClaw의 가이드를 따르세요.
- **백업 필수:** `sandboxes.json` 설정 파일과 `workspace` 폴더의 데이터는 일주일에 한 번씩 백업하는 습관을 들여야 합니다.

## 5. 실전 실습: 사내 지식 비서 테스트

**과제: "사내 보안 가이드라인 답변 에이전트" 운영하기**

1. **RAG 데이터 준비:** 사내 보안 가이드(PDF나 MD 파일)를 준비하여 특정 폴더에 넣습니다.
2. **지식 검색 테스트:** "우리 회사에서 USB 사용 규정은 어떻게 돼?"라고 물어보고, 에이전트가 사내 문서를 근거로 답변하는지 확인합니다.
3. **라우팅 로그 확인:**
   - 간단한 인사: "안녕?" -> 로그에서 L1 모델이 응답했는지 확인.
   - 복잡한 보안 설계 질문 -> 로그에서 L2 모델로 라우팅되었는지 확인.
4. **최적화 확인:** 답변이 너무 길거나 불필요한 사설이 많다면, `guidelines.md`를 수정하여 간결한 답변을 유도해 봅니다.
